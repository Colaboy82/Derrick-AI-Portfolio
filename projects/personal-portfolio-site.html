Personal Portfolio Site

Designed a personal portfolio site, built entirely with AI assistance and hosted on a permanent, shareable URL with zero ongoing cost and automatic redeployment for all future updates.

Tools Used

Tool Role in the workflow
VS Code Code editor where the site is built and previewed
GitHub Copilot AI assistant (free tier) that generates the HTML/CSS/JS from a written prompt
Live Preview / Live Server VS Code extensions used to preview the site locally before publishing
Git Version control tool that tracks and saves changes to the project
GitHub Cloud repository that stores the code and connects to Vercel
Vercel Free hosting platform that auto-deploys the site from GitHub

Experience Gained

AI-assisted development

  • • Writing clear, detailed prompts that translate real content (a resume) into structured, working code
  • • Iterating on AI-generated output through follow-up instructions rather than manual rewrites
  • • Reading and lightly editing generated HTML, CSS, and JavaScript to understand what the code does

Version control

  • • Core Git operations: staging, committing, and pushing changes
  • • Understanding how a local project folder stays linked to a remote GitHub repository, including recovering the initial clone and moving the folder

Deployment and hosting

  • • Setting up continuous deployment, where every GitHub push automatically triggers a live update
  • • Configuring a hosting platform (Vercel) end-to-end: importing a repository, deploying, and managing the live URL
  • • Understanding how metadata like the HTML title tag affects how a link appears when shared

Troubleshooting

  • • Diagnosing and fixing a broken editor-to-repository link after relocating a project folder
  • • Debugging an incorrect live preview and tracing it back to a single line of HTML

Workflow Schema

The project follows a linear continuous deployment pipeline. Each stage feeds directly into the next, and the final two stages repeat automatically for every future update.

Stage Role Output
1. Write Prompt engineer index.html (HTML, CSS, JS in one file)
2. Preview Visual reviewer Local visual check before publishing
3. Save Change committer Committed snapshot of the code
4. Store Repository manager Repository accessible to Vercel
5. Deploy Infrastructure observer Live site at project-name.vercel.app
6. Update Push initiator Future edits auto-redeploy on every push, no manual redeploy needed

Steps Taken

1. Set up the environment

  • • Installed VS Code and the GitHub Copilot extension, signed in with a GitHub account
  • • Created a public GitHub repository to hold the project
  • • Cloned the repository to a local folder and opened it in VS Code

2. Generated the site with AI

  • • Wrote a detailed prompt describing the resume content, sections, and design preferences
  • • Used GitHub Copilot Chat to generate a single index.html file containing HTML, CSS, and JavaScript
  • • Previewed the result locally using the Live Preview extension, refining the design through follow-up prompts

3. Published the code

  • • Staged and committed the finished file in VS Code's Source Control panel
  • • Pushed (synced) the commit to GitHub

4. Deployed with Vercel

  • • Signed in to Vercel with GitHub and imported the repository as a new project
  • • Deployed with default settings, since the site is a single static HTML file requiring no build step
  • • Received a free live URL in the format project-name.vercel.app

5. Maintenance and fixes

  • • Resolved a broken folder link after moving the project directory by reopening the folder in VS Code
  • • Updated the page title tag so shared links display the correct name instead of a generic label
  • • Set up a continuous update routine: edit the file, commit, push, and Vercel redeploys automatically
