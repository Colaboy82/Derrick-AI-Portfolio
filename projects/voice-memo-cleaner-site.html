Voice Memo Transcription App

Live site: voice-memo-cleaner.vercel.app

Built a full-stack web app that takes a voice memo, transcribes it to text using AI speech recognition, removes filler words and verbal tics, and automatically generates a structured summary, key points, and action items — all deployed to a live URL at no cost.

Tools Used

Tool Role in the workflow
VS Code Code editor used to write, run, and debug the entire project
Next.js 14 Full-stack React framework that handles both the frontend UI and backend API routes in one codebase
TypeScript Typed superset of JavaScript that catches bugs at compile time before the code runs
Groq Free AI inference platform that hosts OpenAI's Whisper speech-to-text model for audio transcription
Google Gemini Free large language model API used to remove filler words and generate structured notes from transcripts
Tailwind CSS Utility-first CSS framework used to style the UI with class names instead of separate stylesheets
Vercel Free hosting platform that deploys the app automatically and runs the API routes as serverless functions
Git / GitHub Version control and cloud repository used to store the project and connect to Vercel for deployment

Experience Gained

  • • Calling a large language model API (Google Gemini) with a natural language prompt and parsing the response
  • • Running two independent API calls in parallel using Promise.all() to cut total processing time in half
  • • Diagnosing and fixing a model availability error by switching from whisper-large-v3 to whisper-large-v3-turbo on the free tier

Prompt engineering

  • • Writing precise prompts that produce reliable, consistent output from a language model rather than vague or unpredictable responses
  • • Asking the model to return structured JSON by including the exact field names and shape in the prompt
  • • Handling edge cases where the model wraps JSON output in markdown code fences despite being instructed not to

Full-stack Next.js development

  • • Building serverless API routes using the Next.js App Router, where each file in app/api/ becomes a live endpoint automatically
  • • Separating server-side code (API calls with secret keys) from client-side code (the browser UI) to keep credentials secure
  • • Managing application state in React using useState and useRef hooks to drive the UI through loading, processing, and done stages
  • • Sending files from the browser to a server using FormData and the fetch API

Security and environment management

  • • Storing API keys in .env.local so they are never included in the codebase or sent to the browser

Deployment and troubleshooting

Workflow Schema

The app follows a linear request pipeline. Each stage feeds directly into the next. Stages 3 and 4 run in parallel.

Stage Role Output
1. Upload File receiver Audio file (.mp3, .m4a, .wav, .webm) held in browser memory
2. Transcribe Speech-to-text Raw transcript text returned from Groq Whisper API
3. Clean Filler word removal Polished transcript with um, uh, like, you know removed — runs in parallel with stage 4
4. Notes Note generation JSON object with title, summary, key points, and action items — runs in parallel with stage 3
5. Display UI renderer Four-tab output panel: Cleaned, Summary, Key Points, Action Items

Steps Taken

1. Set up the project

  • • Ran npx create-next-app@latest with TypeScript and Tailwind CSS flags to scaffold the project in VS Code
  • • Installed the Groq SDK and Google Generative AI SDK using npm
  • • Created a .env.local file in the project root to store API keys securely outside the codebase
  • • Signed up for a free Groq account at console.groq.com and a free Gemini key at aistudio.google.com — no credit card required for either

2. Built the transcription API route

  • • Created app/api/transcribe/route.ts as a Next.js serverless function that accepts POST requests
  • • Used req.formData() to unpack the uploaded audio file from the incoming HTTP request
  • • Passed the file to the Groq SDK using groq.audio.transcriptions.create() with the whisper-large-v3-turbo model
  • • Returned the raw transcript text as a JSON response to the frontend

3. Built the filler-word removal route

  • • Created app/api/clean/route.ts to receive the raw transcript as JSON and send it to Gemini
  • • Wrote a detailed prompt that names specific filler words to remove, defines false starts and stutters with examples, and instructs the model not to rephrase or add new content
  • • Called model.generateContent() from the Google Generative AI SDK and extracted the plain text response

4. Built the notes generation route

  • • Created app/api/notes/route.ts to receive the same raw transcript and return structured notes
  • • Wrote a prompt that instructs Gemini to return a JSON object with four fields: title, summary, keyPoints (array), and actionItems (array)
  • • Added defensive cleanup to strip markdown code fences from the response before calling JSON.parse()
  • • Wrapped JSON.parse() in a try/catch block so a malformed model response returns a 500 error instead of crashing the app

5. Built the frontend

  • • Built the main page in app/page.tsx as a client-side React component using the 'use client' directive
  • • Added a drag-and-drop upload zone that triggers a hidden file input when clicked and handles dropped files via onDrop
  • • Called /api/transcribe first, then used Promise.all() to call /api/clean and /api/notes simultaneously
  • • Rendered results in a four-tab panel (Cleaned, Summary, Key Points, Action Items) with a collapsible raw transcript at the bottom
  • • Added a copy-to-clipboard button on the cleaned transcript tab and checkbox inputs on the action items tab

6. Deployed to Vercel

  • • Initialised a Git repository, committed the project, and pushed to GitHub
  • • Installed the Vercel CLI and ran vercel to link the project to a Vercel account
  • • Added GROQ_API_KEY and GEMINI_API_KEY in the Vercel dashboard under Settings -> Environment Variables
  • • Ran vercel --prod to deploy the production build and received a live URL at voice-memo-cleaner.vercel.app

7. Debugging and fixes

  • • Encountered a subscription error on Groq: resolved by switching from whisper-large-v3 to whisper-large-v3-turbo, which is available on the free tier
  • • Encountered Vercel deployment failures: diagnosed by reading build logs in the Vercel dashboard and confirming environment variables were correctly set
  • • Added defensive JSON cleanup after discovering Gemini occasionally wraps output in markdown code fences despite prompt instructions
